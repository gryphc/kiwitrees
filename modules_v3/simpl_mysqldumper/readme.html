ReadMe file

for webtrees

This is a very small, simple module with a very powerful purpose. It provides an ability to include the excellent "MySQLDumper" backup software into your webtrees administration panel options. Throughout webtrees documentation you are very sensibly advised to back up your database at frequent intervals, and especially before any upgrade. If your web host provides a tool like PhpMyAdmin or cPanel you should, if you know how, be able to use that. But for greater flexibility, simplicity and convenience MySQLDumper is hard to beat.
See side panel for MySQLDumper's own description of its features.

I find the ability to automatically email you a zipped copy of the complete backup particularly helpful. To go further, perhaps with some help from your web host, you can set up a "cron job" to make the entire process automated, emailing you a fresh backup every few days or weeks.

Once installed you will see a new link in the module-specific section of the administration panel, bottom left corner, similar to that shown here to the left.

This module will work with any version of webtrees since about 1.2, including .

You also need a working installation of MySQLDumper. This is NOT provided as part of this module. See Installation.
Support for Simpl_add-ons is available only at the Kiwitrees forum pages.

Updates and bug-fixes will be announced on the Simpl_add-ons blog pages, and on Twitter.

  1. Before doing anything with these files, download and install the latest version of MySQLDumper from www.mysqldumper.net. Where you install it is not critical as this module simply requires you to provide a working link to the installation. However, I recommend for simplicity and security that you install it in the same part of your web server as your webtrees files, perhaps in a parallel directory. After installation, access MySQLDumper directly to set it up and to check it is fully functional. There is no point wasting time with this module until you have a working installation of MySQLDumper. [ NOTE: If you have any problems installing or using MySQLDumper please contact them at their website for support. They provide excellent service. www.mysqldumper.net]
  2. Once MySQLDumper is working you can proceed with installing this webtrees module. Unzip the files and upload them to the webtrees/modules_vX/ folder on your web server so that you have a new folder there called "simpl_mysqldumper". (Note: "X" = "3" at the time of writing this, but may change in future releases)
  3. Login to webtrees as administrator, go to AdministrationModules, and find the module. It will be called "Simple_SQL Backup". Check it has a tick for "Enabled"
  4. Finally click SAVE, to complete the installation. Refresh your administration screen ("F5" in most browsers) and you will see the link in the menu, although it will not work yet. See Setup next.
There is just one item requiring setup before this module can be used. Open the file, from this module, called module.php. At line 66 replace the entry "mysqldumper/index.php" with the correct path to the index.php file in your own MySQLDumper installation. Then save the revised file back to your web server.
The changes in this version of the module are non-critical, so if you prefer, you do not NEED to upgrade it for webtrees 1.4.3. It will continue to work normally.

The operational change just reduces the number of files in the module by one. The file admin_mysqldumper.php has been removed as we are now able to perform the same functions within a single module.php file.

If you do want to upgrade, rename your old simpl_mysqldumper folder, upload this one and re-do the Setup instruction. You should not need to re-enable the module. Then check the module is still working and if it is, delete the old renamed folder.
This module has no translation requirements.

MySQLDumper has it's own language files, and you can set your preferred language in it's configuration settings. Any further questions on translation should be addressed directly to that product's developer.
